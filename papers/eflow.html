已有方法的局限性

  • 双重效率瓶颈：视频扩散 Transformer 同时面临两个独立瓶颈——每步 Attention 的二次时空复杂度，以及需要数十步迭代采样。这两个问题往往被分别研究，缺乏统一解决方案。
  • 少步训练不稳定：Solution Flow（从噪声时间步 t 直接映射到时间步 s 的函数）在视频规模上难以稳定训练，因为每步的引导目标计算代价极高。
  • Token Dropping 与训练不一致：现有稀疏 Attention 方法在训练时使用稠密 Attention，推理时使用稀疏 Attention，导致 train-test gap。

本文核心问题

能否从头设计一个统一解决「计算复杂度」和「采样步数」双重瓶颈的视频生成框架，同时保持高生成质量？

⚙️ 核心方法

整体框架

EFlow 建立在 Solution Flow 目标上（直接预测 t→s 的映射），通过三个互补创新同时解决双重瓶颈：Gated Local-Global Attention 降低每步计算量，Path-Drop Guided Training 降低少步训练代价，Mean-Velocity Additivity 正则化保证极少步数下的生成质量。

关键模块

① Gated Local-Global Attention：混合 Attention 块，局部 Attention 处理空间近邻 token，全局 Attention 处理跨帧关键 token。创新点：引入可训练 Gate，在训练时即随机 drop 大量 token（激进 token dropping），推理时同样 drop，从根本上消除 train-test gap。Gate 机制确保 token dropping 下的训练稳定性。
② Path-Drop Guided Training：Solution Flow 训练中，正确的引导目标需要运行完整扩散过程，计算代价极高。Path-Drop 用「随机 drop 部分扩散步」的简化轨迹（弱路径）替代完整引导目标，大幅降低训练成本，同时通过理论分析证明这不会显著损害最终生成质量。
③ Mean-Velocity Additivity 正则化：在极低步数（1-2 步）下，Solution Flow 的预测误差会累积导致质量下降。此正则化约束模型的速度场满足「均值可加性」（mean-velocity additivity），数学上保证极少步采样时的收敛性。

与已有方法的本质区别

现有少步蒸馏方法（如 LCM、DMD）需要预训练好的教师模型；EFlow 是从头训练的统一框架，无需预训练教师。Gated Local-Global Attention 在训练时就集成 token dropping，而非推理时的事后稀疏化。

潜在问题
  • 从头训练需要大量计算资源，对无法从头训练的用户不友好（相比蒸馏方法）。
  • Gated Local-Global Attention 中 Gate 的训练稳定性在超大规模（如 5B+ 参数）模型上是否保持尚不明确。
  • 极少步数（1步）生成质量与全步数方法仍有差距，创意生成场景可能不够理想。
