SpecKV: Adaptive Speculative Decoding with Compression-Aware Gamma Selection

Shikhar Shukla et al. · 2026-05

推测解码 · 自适应 Gamma KV 压缩 · LLM 加速 arXiv 2605.02888 ↗

TL;DR

SpecKV 为推测解码引入了压缩感知的自适应推测长度（γ）控制器。草稿模型的置信度与熵信号与接受率高度相关（约 0.56），轻量 MLP 控制器据此动态选择每步最优 γ，跨 4 类任务 × 3 种 KV 压缩级别均有效。vs 固定 γ=4 提升 56.0%，额外开销仅 0.34ms，代码开源。

研究背景

  • 固定 γ 的局限：推测解码中固定推测长度 γ 无法适应不同任务难度，简单任务下 γ 可以更长，复杂任务下长 γ 会导致大量验证失败
  • KV 压缩对接受率的影响：量化 KV Cache（INT8/NF4）会降低验证模型精度，进而影响接受率，固定 γ 在高压缩下表现更差
  • 任务异质性：代码生成、数学推理、对话、文档摘要等不同任务对 γ 的最优选择差异显著
  • 动态适应需求：同一生成序列中，不同位置的接受率也会随上下文变化，需要逐步动态调整

核心方法

1. 压缩 × 任务 Profiling：

// 离线 Profiling 矩阵 Profile[task_type][γ][compression_level] → acceptance_rate // 4 类任务 × 4 种 γ (1,2,4,8) × 3 压缩级别 (FP16,INT8,NF4) // = 48 个配置的接受率统计

2. 草稿信号 → 接受率预测：发现草稿模型的 top-1 置信度和分布熵与验证接受率强相关（r≈0.56），用这两个轻量信号作为在线预测特征。

// MLP 控制器（极小参数量） γ* = MLP([confidence, entropy, task_type, compression_level]) // 输入：草稿置信度、预测熵、任务类型嵌入、当前压缩级别 // 输出：最优推测长度 γ ∈ {1, 2, 4, 8} // 开销：仅 0.34ms/step

3. 在线自适应推测：每个解码步骤调用 MLP 控制器获取当前最优 γ，无需重新 profile，实现逐步动态适应。

实验结果

  • vs 固定 γ=4：整体加速提升 56.0%
  • 控制器开销：每步仅 0.34ms，相对于推测解码本身可忽略不计
  • 置信度-接受率相关性：Pearson r ≈ 0.56，验证了信号有效性
  • NF4 压缩场景：自适应 γ 显著优于固定 γ，压缩导致的接受率下降被有效补偿
  • 跨任务泛化：4 类任务（代码/数学/对话/摘要）均一致受益
  • 开源：代码公开，可直接集成到现有推测解码框架

核心贡献

  1. 首次将 KV 压缩级别纳入推测解码的 γ 选择决策，解决压缩与推测的联合优化问题
  2. 发现草稿置信度/熵是接受率的有效预测信号，为自适应控制提供轻量特征
  3. 轻量 MLP 控制器仅 0.34ms 开销实现逐步动态 γ 选择
  4. 系统性 profiling 框架（任务×长度×压缩）为后续研究提供基准数据集

局限性

  • 离线 profiling 阶段需要对 48 种配置进行统计，初始设置成本较高
  • 任务类型分类（4 类）较粗糙，细粒度任务可能需要更精细的分类
  • 置信度-接受率相关性约 0.56，仍有约 44% 的方差未被解释
  • 目前针对草稿模型和验证模型固定配对，不同草稿模型需重新 profile
