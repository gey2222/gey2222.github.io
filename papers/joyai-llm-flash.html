已有方法的局限性

  • 中等规模模型 token 效率低：sub-50B 模型在保持高性能的同时往往推理时输出冗余 token 过多，实际推理成本高。
  • RL 信任域约束不够精细：现有 PPO/GRPO 等算法对信任域的约束为全局性的，在多任务分布下容易过拟合某类偏好而损害其他维度。
  • 训练推理协同设计不足：MoE 模型在 QAT 量化与 MTP 多 token 预测联合优化方面缺乏成熟方案，量化部署后推理协同能力容易退化。

本文核心问题

如何在 sub-50B 规模内同时实现：接近更大模型的推理性能、高 token 效率（避免冗余推理链）、灵活的 thinking/non-thinking 模式切换？

⚙️ 核心方法

整体框架

JoyAI-LLM Flash 是一个完整的 LLM 研发体系。MoE 架构实现参数规模与计算成本解耦；20T token 大规模预训练提供知识基础；SFT → DPO → RL 完整后训练流水线，RL 阶段使用 FiberPO；推理部署通过 QAT 实现低比特量化，且与 MTP 联合训练以维持量化后推理能力。

关键模块

① MoE 极高稀疏架构：48B 总参数，推理时仅激活 2.7B，稀疏比显著高于同量级竞品（如 DeepSeek-V2 Lite 等）。专家级知识分工使模型在低激活参数下保留大容量知识。
② FiberPO 强化学习：受纤维束理论（Fiber Bundle Theory）启发，将 RL 信任域约束分解为「全局约束」（控制整体策略分布漂移）和「局部约束」（针对不同任务子空间的细粒度调节），两层约束协同实现多尺度稳定性控制，避免多任务 RL 对齐的过拟合问题。
③ 稠密 MTP + QAT 联合训练：多 token 预测头提供额外监督信号提升训练样本效率；QAT 与 MTP 联合优化确保量化模型保留多步推理协同能力；thinking/non-thinking 双模式通过特殊控制 token 切换。

关键公式（FiberPO 约束）

L_FiberPO = L_RL + λ_g · D_KL(π_global ‖ π_ref)
+ Σᵢ λᵢ · D_KL(π_local⁽ⁱ⁾ ‖ π_ref⁽ⁱ⁾)
其中全局项控制整体分布偏移，局部项针对第 i 个任务纤维独立调节

与已有方法的本质区别

与 DeepSeek-V3/Qwen3 等同量级模型相比，核心差异在于 FiberPO 的信任域分解——将全局 KL 散度约束与局部任务级约束分离，理论上在多任务 RL 对齐中具有更好优化稳定性。MTP 与 QAT 的联合训练也是较新的工程实践。

✨ 贡献点

1

2
实现 MTP 与 QAT 的联合训练框架，确保量化部署后模型保留多步推理的协同能力——工程与算法的协同创新。
3
