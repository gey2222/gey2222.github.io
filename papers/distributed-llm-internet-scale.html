Distributed Generative Inference of LLM at Internet Scales with Multi-Dimensional Communication Optimization

Jiu Chen, Shuangyan Yang, Xu Xiong, Hexiao Duan et al. · 2026-04

LLM · 去中心化推理 · 系统 互联网规模 · 多维通信优化 · 异构节点 arXiv 2504.16189 ↗

TL;DR

去中心化 LLM 推理将计算分散到互联网异构节点，但高延迟和异质性让现有系统难以高效运行。本文提出多维通信优化框架，自适应配置张量/流水线/序列并行，相比 Petals 吞吐量提升 2.6x，首 token 延迟降低 45%。

研究背景

去中心化推理的潜力与挑战：

  • 潜力：互联网上存在大量闲置算力（个人 GPU、消费级设备），聚合可降低推理成本
  • 网络延迟：数据中心内 GPU 通信延迟 μs 级，互联网节点间延迟 10-100ms，差距达万倍
  • 节点异质性：不同节点内存、计算能力、网络带宽差异巨大，静态负载分配效率低
  • 现有系统局限：Petals 仅支持流水线并行，无法利用带宽-延迟多样性

核心方法

多维并行自适应配置：

  • 张量并行（TP）：适合同一局域网内低延迟节点组，充分利用高带宽
  • 流水线并行（PP）：适合跨地域高延迟节点链，隐藏传输延迟
  • 序列并行（SP）：长序列时分割序列维度，减少单节点内存压力
优化目标：argmin_{TP,PP,SP} E[Latency(request)]
约束：Σ Memory_i ≥ Model_size，∀i Bandwidth_i 已知

自适应调度器：
1. 节点探测：实时测量延迟、带宽、计算能力
2. 拓扑聚类：将低延迟节点聚为 TP 组
3. TP 组间做 PP，长序列追加 SP
4. 批调度：感知异质性分配不同批大小

实验结果

  • 吞吐量：32 个异构互联网节点，比 Petals 提升 2.6x，比单纯 PP 提升 1.7x
  • 首 token 延迟：P50 延迟降低 45%，P99 延迟降低 38%（受网络抖动限制）
  • 节点利用率：CPU 等待时间从 Petals 的 42% 降至 8%
  • 容错性：节点故障恢复时间 <500ms，不中断服务

核心贡献

  1. 首个支持互联网规模多维并行自适应配置的去中心化推理框架
  2. 节点感知调度器，动态优化 TP/PP/SP 配置
  3. 通信-计算重叠消除跨节点等待
  4. 在真实互联网节点上验证，有工程实用价值

局限性

  • 互联网节点不可信任，安全性和隐私保护尚未充分解决
  • 拓扑探测和配置搜索存在启动开销，不适合极短请求
  • 网络抖动大时 P99 延迟仍难以预测，不适合实时交互场景
