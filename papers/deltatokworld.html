📌 研究背景与动机

研究方向

视频生成世界模型高效 Token 化

已有方法的局限性

  • 判别式 vs 生成式的根本矛盾：判别式世界模型（预测特征均值）计算效率高但无法生成多样化未来；生成式世界模型能生成多样未来但计算成本极高。
  • 时空 Token 冗余：传统方法将视频表示为时空 token 网格，对 512×512 帧产生数千个 token，序列极长，Transformer 的二次复杂度成为瓶颈。
  • VFM 特征利用不足：在视觉基础模型（VFM）特征空间预测视频虽然参数少，但现有方案仍是判别式的，无法生成多样化预测。

本文核心问题

能否设计一种极致压缩的视频表示，使生成式世界模型在参数和计算上与判别式方法相当，同时保留生成多样化未来的能力？

⚙️ 核心方法

整体框架

整个系统分两部分：DeltaTok（tokenizer）将视频压缩为 delta token 序列；DeltaWorld（world model）在这个极简表示上进行生成式未来预测。

关键模块

① DeltaTok — Delta Token 化：对视频中相邻帧，先用预训练 VFM 提取特征，再计算帧间特征差值（delta = VFM(frame_t) - VFM(frame_{t-1})），最后将这个差值向量编码为单个连续 token。关键洞察：相邻帧差值捕获了「发生了什么变化」而非「整个场景是什么」，信息密度极高，同时消除了帧内大量冗余信息。结果：512×512 视频从 ~1024 个空间 token/帧 压缩为 1 个 delta token/帧，token 减少 1024x。
② DeltaWorld — 生成式世界模型：在 delta token 一维时间序列上训练自回归生成模型，预测未来 delta token 序列。支持多假设并行训练（Multi-Hypothesis Training）：并行采样生成 N 种未来 delta token 序列，与 ground truth 计算最优匹配损失（只监督最优的一个），强迫模型学习多样化未来分布而非均值预测。推理时单次前向传播即可输出多种不同未来预测。

关键公式

Delta Token 定义：δ_t = Encoder(VFM(I_t) - VFM(I_{t-1}))
Multi-Hypothesis Training Loss：
L = min_{i∈[N]} ‖δ̂_t^(i) - δ_t‖²
（N 个并行预测中只监督与 GT 最近的一个）

与已有方法的本质区别

现有生成式世界模型（如 UniSim、DreamerV3）在像素或 latent 空间生成完整帧，token 数量极多。DeltaTok 的本质创新是：把「预测完整未来帧」转变为「预测帧间变化量」，将生成问题从高维空间降至极低维的时间序列，同时用 VFM 特征保证语义质量。

✨ 贡献点

1
Delta token 的设计——用「帧差 VFM 特征」替代「完整帧表示」，是真正新颖的压缩思路，将视频从 3D 时空降至 1D 时间序列。
2
Multi-Hypothesis Training 方案——在极简 token 表示上实现多样化未来生成，解决了生成式 vs 判别式的根本矛盾。
3
实现了生成式世界模型在参数和计算上向判别式方法量级的跨越式提升（35x / 2000x），打开了实用化世界模拟器的窗口。

潜在问题
  • Delta token 只编码帧间变化，丢失了帧内绝对内容信息——对于「从零初始化」的长程生成，误差可能累积。
  • 高度依赖 VFM 特征空间的质量，对 VFM 不擅长的场景（如暗光、遮挡）可能表现下降。
  • 目前仅在密集预测任务上验证，对需要高分辨率视觉细节的创意视频生成场景适用性待探索。
