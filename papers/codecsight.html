CodecSight: Leveraging Video Codec Signals for Efficient Streaming VLM Inference

Yulin Zou, Yan Chen, Wenyan Chen et al. · 2026-04

视频流分析 · VLM 编解码器信号 · 端到端优化 arXiv: 2604.06036 ↗

TL;DR

视频编解码器天然输出运动向量和残差信号——这是免费的帧间冗余信息。CodecSight 将编解码器信号同时用于优化 ViT 和 LLM 两个阶段，在视频流 VLM 推理上实现端到端加速，无需离线训练或在线冗余检测开销。

研究背景

视频流分析（如安防监控、直播内容理解）是 VLM 服务的重要场景，但高频推理带来巨大计算成本。

现有冗余利用方法的局限：

  • 局部优化：只优化 ViT 或 LLM 其中一个阶段，遗留大量优化空间
  • 冗余检测开销大：需要离线 profiling 或在线运行额外的相似度计算模型，不适合实时流
  • 流内容自适应性差：静态规则（每 N 帧重算）无法适应动态场景变化

关键洞察：视频编解码器（H.264/H.265）在压缩过程中已经计算了运动向量（Motion Vectors）和残差（Residuals），这些信号精确量化了帧间差异——完全可以作为"免费"的冗余检测信号直接使用！

核心方法

编解码器信号双级利用：

  • ViT 阶段优化：
    • 利用运动向量识别静态区域（运动向量近零）的 patch，跳过这些 patch 的 ViT 重新编码
    • 残差信号幅值小的区域直接复用上一帧的 ViT 特征，零额外计算
  • LLM 阶段优化：
    • 基于编解码器宏块运动统计估计帧级冗余度，对高冗余帧跳过 LLM 处理，复用上帧输出
    • 对关键帧（场景切换、大运动）触发完整 LLM 推理
  • 自适应阈值：运动向量幅值和残差能量的动态阈值自动适应不同视频内容（运动激烈 vs 静态场景）
  • 零额外开销：编解码器信号在解码阶段天然产生，无需任何额外模型或计算

实验结果

  • 视频流 QA 任务：端到端吞吐量提升 2.4-3.1x（不同视频内容），精度下降 <1%
  • ViT 阶段：平均跳过 55% 的 patch 重编码，对监控类静态背景视频达 72%
  • LLM 阶段：平均跳过 40% 帧的 LLM 推理，动作片类视频降至 25%
  • 与逐帧全量推理相比，GPU 内存占用降低 45%，时延 P99 降低 60%

核心贡献

  1. 开创性地将视频编解码器信号用作 VLM 推理冗余检测的零成本信号源
  2. 首个端到端同时优化 ViT 和 LLM 两阶段的视频流 VLM 服务框架
  3. 完全在线、自适应，无需离线 profiling 或训练，直接部署于任意视频流场景

局限性

  • 依赖编码视频格式（H.264/H.265），对原始帧（RAW）或 JPEG 序列不适用
  • 编解码器信号粒度（宏块级）较粗，可能漏检局部细粒度变化
  • 对快速场景切换（频繁关键帧）的视频加速效果有限
