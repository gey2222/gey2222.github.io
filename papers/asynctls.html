AsyncTLS: Efficient Generative LLM Inference with Asynchronous Two-level Sparse Attention

Yuxuan Hu, Jianchao Tan, Jiaqi Zhang et al. · 2026-04

LLM · 稀疏注意力 长上下文 · 异步卸载 · KV Cache arXiv: 2604.07815 ↗

TL;DR

分层稀疏注意力（粗粒度块过滤 + 细粒度 token 选择）+ 异步卸载引擎（计算与传输重叠），在 Qwen3、GLM-4.7-Flash 上同时支持 GQA 和 MLA 架构，精度接近全注意力，内存和计算显著降低。

研究背景

长上下文 LLM 推理面临双重挑战：

  1. 二次方注意力复杂度：128k 上下文时注意力计算约占总推理时间 60%+
  2. KV Cache 内存爆炸：单次 128k 推理的 KV Cache 可达数十 GB

现有解法的 trade-off：

  • Token 级稀疏注意力：精度高，但为每个 query 逐 token 建索引开销大（额外 10-20% 延迟）
  • 块级稀疏注意力：索引简单高效，但块内粒度粗，重要 token 可能因所在块整体不重要而被丢弃
AsyncTLS 融合两者优势，并通过异步卸载解决内存问题。

核心方法

两级分层注意力（Two-Level Sparse Attention）：

  • 第一级（块过滤）：用 query 向量对 KV 块进行粗粒度相关性打分，快速过滤掉大量无关块（保留 top-20% 块）
  • 第二级（Token 精选）：在保留块内用精确注意力分数做细粒度 token 选择，只对高得分 token 计算完整注意力

异步卸载引擎（Asynchronous Offloading）：

  • 利用时间局部性（temporal locality）预测下次需要的 KV 块，提前异步从 CPU 预取到 GPU
  • KV 传输与 GPU 计算完全重叠，卸载延迟几乎为零
  • 支持 GQA（分组查询注意力）和 MLA（多头潜在注意力）等高效注意力变体
Stage 1: score_block(q, K_block) = max_t softmax(q·K_block[t])
Stage 2: sparse_attn(q, top_tokens) = softmax(q·K_selected) · V_selected
Async: prefetch(next_blocks) || compute(current_blocks)

实验结果

  • Qwen3-72B 128k 上下文：内存占用降低 55%，注意力计算加速 3.2x，LongBench 精度相比全注意力下降 <1%
  • GLM-4.7-Flash MLA 架构：同等精度下推理延迟降低 45%，支持 2x 更长上下文窗口
  • 与纯块级方法相比，AsyncTLS 在 Multi-Doc QA 上精度提升 4.5 个百分点
  • 异步预取将 CPU→GPU 传输延迟占比从 20% 降至 <3%

核心贡献

  1. 创新性地将块过滤与 token 精选融合为两级稀疏注意力，突破精度-效率 trade-off
  2. 设计基于时间局部性的异步 KV 预取引擎，实现计算-传输完全重叠
  3. 首次在 MLA 架构上验证分层稀疏注意力的有效性，覆盖主流 LLM 注意力变体

局限性

  • 两级过滤引入额外内存访问，在极低上下文长度（<8k）下开销不合算
  • 时间局部性预取对访问模式规律性有一定假设，对高度随机访问场景预取效果下降
  • 在超长上下文（>1M tokens）下需要配合磁盘级卸载，架构需进一步扩展
