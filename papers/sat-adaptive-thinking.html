SAT: Balancing Reasoning Accuracy and Efficiency with Stepwise Adaptive Thinking

Weiyang Huang, Xuefeng Bai, Kehai Chen et al. · 2026-04

LLM · 推理效率 反过度思考 · FSM · 步骤剪枝 arXiv: 2604.07922 ↗

TL;DR

大型推理模型"过度思考"普遍存在。SAT 将推理建模为有限状态机（FSM），通过步骤级难度感知剪枝动态在慢/正常/快/跳过四种思考模式间切换，减少 25-40% token 用量，同时维持推理准确率。

研究背景

以 DeepSeek-R1、o1 为代表的大型推理模型（LRM）通过长链式推理（Long CoT）实现了数学、编程等复杂任务的突破。然而，过度思考（overthinking）是其显著缺陷：

  • 简单问题也生成数千 token 的冗余推理链
  • Token 消耗与 API 成本直接挂钩（o1 系列每次推理成本比 GPT-4o 高 5-10x）
  • 推理链中充斥不必要的"自我反省"、重复验证等冗余步骤

现有解决方案多为全局截断（直接限制总 token 数），破坏推理链的逻辑完整性；或训练时干预，需大量标注数据。

核心方法

有限状态机（FSM）推理框架，定义四种思考模式：

状态定义：
🐌 SLOW: 全深度推理（复杂子问题）
📝 NORMAL: 标准 CoT 步骤
⚡ FAST: 压缩单步推导（已知规律）
⏭️ SKIP: 直接跳过（显然结论）
  • 步骤难度评分：轻量级过程奖励模型（PRM）对每个推理步骤评估难度（0-1 分），决定当前步骤进入哪种思考状态
  • FSM 状态转移：根据 PRM 得分和历史推理链质量，动态在四种模式间转换，高难度步骤启用 SLOW 模式，低难度启用 FAST/SKIP
  • 逻辑完整性保护：通过依赖图分析，确保被跳过/压缩的步骤不影响后续推理步骤的逻辑依赖
  • 无需重训练：SAT 作为推理时框架，直接应用于现有 LRM（DeepSeek-R1、QwQ 等），仅需轻量 PRM 校准
mode(step_t) = FSM_transition(PRM_score(step_t), history)
token_budget(step_t) = mode_budget[mode(step_t)]
accept step_t if: dependency_check(step_t, prior_steps) = True

实验结果

  • DeepSeek-R1 上：MATH-500 准确率保持 92.1%（全量 92.8%），token 用量减少 38%
  • QwQ-32B 上：AMC23 准确率 85.4%（全量 86.1%），token 用量减少 31%
  • SKIP 模式启用率：约 15% 步骤可直接跳过，FAST 模式约 35%，显示大量推理冗余
  • 相比全局 token 限制（截断法），SAT 在相同 token 预算下准确率高 6-12 个百分点

核心贡献

  1. 首次将 LRM 推理建模为有限状态机，实现步骤级粒度的动态难度自适应控制
  2. 轻量 PRM + 依赖图机制，保证推理剪枝的逻辑完整性
  3. 无需重训练的即插即用框架，对主流 LRM 均有效

局限性

  • PRM 校准质量影响剪枝效果，低质量 PRM 可能误剪关键步骤
  • 在需要深度验证的证明类任务（如 IMO 级别）上剪枝空间有限
  • 依赖图分析增加少量额外计算（约 5%），在极短推理链上不合算
