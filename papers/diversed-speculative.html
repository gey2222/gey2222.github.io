DIVERSED: Relaxed Speculative Decoding via Dynamic Ensemble Verification

Ziyi Wang, Siva Rajesh Kasa, Ankith M S · 2026-04

LLM · 推测解码 推理加速 · 动态集成验证 arXiv: 2604.07622 ↗

TL;DR

标准推测解码的刚性验证拒绝太多合理 token。DIVERSED 通过动态集成多个草稿模型进行放宽验证，在保证输出质量的前提下大幅提升 token 接受率，从而获得更高端到端加速比。

研究背景

推测解码（Speculative Decoding）是 LLM 推理加速的重要技术：用小草稿模型并行生成多个候选 token，再用目标大模型一次性批量验证。

核心瓶颈在于验证步骤的刚性：标准方法要求接受的 token 分布必须与目标模型严格匹配（KL divergence = 0）。这导致即使语义上完全合理的草稿 token 也因概率分布轻微差异而被拒绝，token 接受率（acceptance rate）偏低，加速比大打折扣。

核心方法

动态集成验证（Dynamic Ensemble Verification）：

P_ensemble(x) = Σk wk(context) · P_draft_k(x)
Accept token t if: P_ensemble(t) / P_target(t) ≥ threshold(dynamic)
  • 多草稿模型集成：维护多个轻量草稿模型，根据上下文动态加权组合其输出分布，形成更接近目标模型的集成分布
  • 放宽接受准则：基于集成分布与目标分布的近似程度动态调整接受阈值，允许接受高置信度的近似匹配 token
  • 质量保障机制：通过统计置信界限确保放宽验证不导致输出质量退化，仅在集成分布高置信时放宽阈值
  • 动态权重调整：监控各草稿模型的历史接受率，在线更新集成权重，偏向表现更好的草稿模型

实验结果

  • 在 Llama-3 70B 推理上，DIVERSED 相比标准推测解码 token 接受率提升 15-25%
  • 端到端吞吐量（tokens/s）比标准推测解码提升 1.3-1.8x，在长序列生成任务上提升更显著
  • 输出质量（ROUGE、BERTScore）与贪心解码基线相当，无明显退化
  • 在代码生成（HumanEval）任务上尤其有效，接受率提升超过 30%

核心贡献

  1. 识别标准推测解码中刚性验证的核心瓶颈，提出动态集成放宽验证框架
  2. 证明多草稿模型集成能够构建更接近目标分布的代理，为放宽验证提供理论支撑
  3. 设计在线动态权重调整机制，使 DIVERSED 无需离线校准即可适应不同任务

局限性

  • 维护多个草稿模型增加内存开销，对 GPU 内存有更高要求
  • 动态权重调整的在线计算有少量额外开销（约 2-3%）
  • 在极短序列生成（<50 tokens）场景下加速效果有限
