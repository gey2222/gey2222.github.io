已有方法的局限性

  • 过度推理（Overthinking）：大推理模型（如 o1、R1）在简单问题上也会生成极长的推理链，造成大量计算浪费，甚至反而降低性能。
  • 现有早停方法过于简单：基于步数阈值或固定 token 预算的截断策略无法感知推理质量，可能在关键步骤时截断。
  • 缺乏对推理过程内部动态的理解：此前工作未深入分析正确与错误推理轨迹在中间答案置信度上的差异规律。

本文核心问题

能否通过监控推理过程中中间答案的置信度变化，自适应地判断推理何时已经「收敛」，从而安全地提前终止？

⚙️ 核心方法

核心观察（经验发现）

通过大规模实验分析正确与错误推理轨迹的中间置信度动态，发现两类规律性差异：正确轨迹往往在早期即出现高置信度的稳定中间答案，且置信度曲线单调上升后趋于平稳；错误轨迹则置信度动态混乱，长时间处于低置信或频繁波动状态。这个观察是整个方法的基础。

关键模块

① 中间答案置信度监控：在推理链生成过程中，定期（每 K 步）提取当前「中间答案」并计算模型对该答案的置信度（通过 softmax 概率或 token 概率估计）。
② 置信度动态早停判决（CoDE-Stop）：设计基于置信度时序动态的早停判决规则——当检测到「连续多步中间答案一致且置信度超过阈值」的稳定模式时，触发早停，直接输出当前中间答案作为最终答案。无需额外训练，只需设置置信度阈值和稳定步数两个超参数。

与已有方法的本质区别

现有方法基于外部指标（步数、token 数）截断推理，而 CoDE-Stop 基于推理过程内部的质量信号（置信度动态）判断是否可以安全终止。本质上是把「什么时候该停」的决策权从用户（固定阈值）交给模型自身的内部状态。

潜在问题
  • 假阳性风险：对于需要「后悔」（backtracking）的推理任务，早期高置信答案可能是错误的自信，提前停止会导致错误。
  • 置信度阈值的鲁棒性：最优阈值可能因模型、任务类型不同而变化，需要校准数据集。
  • 对需要验证答案正确性的任务（如数学证明）可能存在过早停止的风险。
