DASH-KV: Accelerating Long-Context LLM Inference via Asymmetric KV Cache Hashing

Jinyu Guo, Zhihan Zhang, Yutong Li, Jiehui Xie et al. · 2026-04

LLM · KV Cache · 长上下文 非对称哈希 · 内存压缩 · 推理加速 arXiv 2504.15422 ↗

TL;DR

标准注意力的二次复杂度严重制约长上下文推理，DASH-KV 提出非对称 KV Cache 哈希方案——对 Key 和 Value 采用不同精度的局部敏感哈希策略，在保留 99.5%+ 生成质量的前提下，将 128k 长文档 KV Cache 内存降低 70%，推理吞吐量提升 2.8x。

研究背景

长上下文 LLM 推理面临两大挑战：

  • 计算瓶颈：自注意力复杂度 O(n²)，128k token 时每步注意力开销极大
  • 内存瓶颈：KV Cache 随序列长度线性增长，70B 模型 128k 上下文占用 >100GB 显存
  • 现有方案不足：均匀压缩（量化/剪枝）损失质量；稀疏注意力（如 H2O、StreamingLLM）依赖固定模式，无法自适应内容

核心洞察：Key 用于相似性检索（语义敏感），Value 承载实际信息（数值精度敏感）——两者对精度的需求天然不对称，应区别处理。

核心方法

非对称哈希策略：

  • Key 侧：局部敏感哈希（LSH）将相似 Key 映射到同一桶，用桶代表代替原始 Key 检索，大幅减少精确匹配开销
  • Value 侧：基于 Key 相似性聚类后对同簇 Value 进行加权合并，保留信息内容同时压缩表示
DASH-KV 注意力计算：
1. LSH(Q, K) → 桶分配 B，时间 O(n log n)
2. 每查询 Q_i 只与同桶 K 精确计算注意力分数
3. 桶内 Value 加权合并：V̂_b = Σ(softmax(QK^T/√d) · V)
4. 非对称精度：Key 哈希 INT8，Value 合并 FP16

有效注意力长度：从 n 降至 O(n/B)，B = 桶数

自适应桶大小：根据序列长度和查询分布动态调整桶粒度，避免过度压缩关键 token。

实验结果

  • 内存效率：LLaMA-3-70B 128k 上下文，KV Cache 内存从 145GB 降至 43GB（降低 70%）
  • 推理速度：吞吐量提升 2.8x，首 token 延迟降低 55%
  • 生成质量：RULER（长文档 QA）得分 Full Attention 的 99.5%+，PASSKEY Retrieval 100% 准确
  • 对比基线：同等压缩率下比 H2O 质量高 2.1 分（RULER），比 SnapKV 高 1.4 分

核心贡献

  1. 首次提出 Key 和 Value 非对称哈希压缩原则，揭示 KV 不对称信息特性
  2. 设计高效 LSH 驱动的稀疏注意力，将复杂度从 O(n²) 降至 O(n log n)
  3. 在主流长文档 benchmark 上实现质量-效率最优 Pareto 前沿
  4. 方案与 FlashAttention、GQA 等工程优化正交可叠加

局限性

  • 哈希碰撞在极端稀疏查询场景下可能遗漏关键 token（如 needle-in-a-haystack 极端任务）
  • LSH 桶构建增加预处理开销，短序列（<8k）收益有限
  • 动态桶大小调整增加实现复杂度，GPU kernel 优化仍有空间
