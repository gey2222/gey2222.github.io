Salca: A Sparsity-Aware Hardware Accelerator for Efficient Long-Context Attention Decoding

Wang Fan, Wei Cao, Xi Zha, Kedi Ma, MingQian Sun, Jialin Chen, Fengzhe Zhang, Fan Zhang · 2026-04

LLM · 硬件加速 · 长上下文 稀疏注意力 · ASIC · 高带宽内存 arXiv 2504.19480 ↗

TL;DR

长上下文 LLM 解码阶段 KV Cache 访问是核心带宽瓶颈，现有 GPU/NPU 对非规则稀疏访问支持差。Salca 设计专用硬件加速器支持列稀疏 KV 访问模式，配合软硬件协同稀疏预测，128k 上下文解码吞吐量比 A100 高 4.3x，功耗降低 62%。

研究背景

  • 带宽瓶颈本质：LLM 解码是内存密集型（memory-bound），每生成 1 token 需要从 DRAM 读取完整 KV Cache；128k 上下文下 KV Cache 达数十 GB
  • 稀疏注意力的困境：算法侧已证明 70%+ 的注意力权重接近 0，但 GPU 不支持高效稀疏内存访问，实际加速不足 1.5x
  • 硬件机会：ASIC 专用设计可原生支持稀疏地址生成和不规则访存，消除 GPU 的通用性开销

核心方法

Salca 硬件架构：

  • 稀疏 KV 访问引擎：专用 DMA 控制器支持列稀疏 KV 地址模式（由稀疏掩码驱动），避免 gather/scatter 的额外开销
  • 双缓冲预取：预测下一时间步的稀疏 Pattern，提前预取 KV 数据，隐藏 DRAM 延迟
  • 软硬件协同：配套在线稀疏预测算法直接输出 Salca 原生地址格式（4-bit 列索引编码），零格式转换开销
稀疏 KV 访问模式：
对每个 query q_i，选择 Top-K KV 对（K = n × sparsity）
Salca 地址生成：addr(k_j) = base + j × stride，j ∈ sparse_mask

吞吐量公式：Throughput = BW / (K × d_head × bytes_per_elem)
K 减少 80% → 吞吐量近线性提升

实验结果

  • 吞吐量：128k 上下文，Llama-3-70B 解码，Salca vs. A100 = 4.3x（1840 token/s vs. 428 token/s）
  • 功耗：62% 降低（Salca 45W vs. A100 SXM 400W at same throughput）
  • 面积：28nm 工艺 28mm²，成本大幅低于 GPU
  • 生成质量：稀疏率 80%，RULER 长文档评估 99.1% vs. 全注意力基线

核心贡献

  1. 首个专为稀疏 LLM 注意力解码设计的 ASIC 加速器 Salca
  2. 软硬件协同稀疏预测-执行流水线，零格式转换开销
  3. 双缓冲预取完全隐藏 DRAM 访问延迟
  4. 在 28nm 工艺实现 4.3x vs. A100 的吞吐量，能效比 GPU 高 10x+

局限性

  • ASIC 设计固化，无法适应未来 Transformer 架构变化（如 MLA、GQA 等）
  • 仅覆盖解码阶段，预填充（Prefill）阶段仍需 GPU/NPU 协作
  • 批量推理（Batch > 1）效益下降，适合单请求低延迟场景
