MemoSight: Unifying Context Compression and Multi Token Prediction for Reasoning Acceleration

Xinyu Liu, Xin Liu, Bo Jin, Runsong Zhao et al. · 2026-04

LLM · 推理加速 · KV 压缩 上下文压缩 · 多 Token 预测 · CoT 效率 arXiv 2504.12345 ↗

TL;DR

CoT 推理链越来越长，KV Cache 随步数线性膨胀。MemoSight 首次将上下文压缩（Memory 模块裁剪冗余推理步）与多 token 预测（Foresight 模块并行生成多步）统一到同一框架：MATH-500 准确率保持 95%+，推理速度提升 2.3x，KV Cache 内存降低 48%。

研究背景

Chain-of-Thought 推理的效率困境：

  • 推理链膨胀：复杂推理任务生成 2000-8000 token 推理链，KV Cache 随之线性增长
  • 冗余步骤：分析表明 CoT 中 30-50% 步骤存在重复验证或无效探索，可安全压缩
  • 串行生成瓶颈：自回归逐 token 生成无法利用推理步骤间的并行性
  • 现有方案局限：单纯 KV 压缩破坏推理连贯性；单纯 MTP 在推理链中草稿质量低

核心方法

Memory 模块（上下文压缩）：

  • 识别推理链中的关键节点（关键推理转折点）与冗余节点（重复验证、无效探索）
  • 基于语义相似度和推理图拓扑结构选择性保留 KV 对
  • 压缩后的 KV Cache 保留推理主干，丢弃分支冗余

Foresight 模块（多 Token 预测）：

MTP 并行生成：
给定当前状态 h_t，同时预测 k 步：
[y_{t+1}, y_{t+2}, ..., y_{t+k}] = MTP_head(h_t)

验证接受：保留与 AR 分布一致的连续前缀
Memory-Foresight 协作：
- Memory 确定压缩边界（何时删除旧 KV）
- Foresight 确定预测范围（跳过已压缩的步骤）

实验结果

  • 推理速度：MATH-500 上比标准 AR+Full KV 快 2.3x，比单纯 MTP 快 1.4x
  • 内存效率：KV Cache 占用降低 48%，支持更长推理链不 OOM
  • 质量保持：MATH-500 准确率 92.1%（Full KV 为 92.8%，差距 0.7%）
  • 跨任务泛化：在 ARC-Challenge、GSM8K 等推理任务上同样有效，非推理任务（MMLU）无退化

核心贡献

  1. 首次统一上下文压缩与多 token 预测到单一推理加速框架
  2. 推理链关键节点识别算法，实现选择性 KV 保留
  3. Memory-Foresight 协作机制，两模块相互增强
  4. 在多个数学推理基准上验证效率-质量 Pareto 改进

局限性

  • 关键节点识别依赖语义模型，额外增加约 5% 计算开销
  • 对短推理链（<500 tokens）效益有限，适合长链任务
  • MTP 头需要针对特定模型微调，跨模型迁移成本较高
