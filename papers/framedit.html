FrameDiT: Diffusion Transformer with Frame-Level Matrix Attention for Efficient Video Generation

FrameDiT：基于帧级矩阵注意力的高效视频生成扩散 Transformer
Minh Khoa Le, Kien Do, Duc Thanh Nguyen, Truyen Tran

TL;DR提出将整帧作为矩阵进行帧间注意力计算的 Matrix Attention 机制，在帧间而非 token 间建模时序依赖，打破 Full 3D Attention（强但贵）与 Local Factorized Attention（快但弱）的效率-质量 trade-off，在多 benchmark 上达到 SOTA。

📌 研究背景与动机

研究方向

视频生成Diffusion Transformer时序建模

已有方法的局限性

  • Full 3D Attention：在所有时空 token 之间计算注意力，能力最强（捕获全局时序依赖），但复杂度为 O((T·H·W)²)，对长视频计算代价极高。
  • Local Factorized Attention：将时序和空间 Attention 分解，或限制 Attention 在局部窗口内，大幅降低复杂度，但对大运动幅度和长时依赖的建模能力受限，视频时序一致性差。
  • 缺乏帧级抽象：现有方法都在 token 级别进行时序建模，没有利用「帧」作为语义单元的自然结构。

本文核心问题

能否设计一种时序注意力机制，在计算效率接近 Local Factorized Attention 的同时，保留 Full 3D Attention 的全局时序建模能力？

⚙️ 核心方法

整体框架

Matrix Attention 的核心思想：将视频中的每一帧视为一个矩阵（H×W 的特征矩阵），在「帧间」而非「token 间」计算注意力。通过矩阵原生操作（矩阵乘法）生成帧级 Q/K/V，并在帧维度上进行 Attention，实现全局时序建模同时大幅降低复杂度。

关键模块

① Matrix Attention 核心机制：对于视频序列，将每帧特征图（H×W×C）视为矩阵。通过左右矩阵乘法（而非 token 拼接）生成帧级的 Query、Key、Value 矩阵，在帧间（T 维度）而非 token 间进行注意力计算。复杂度从 O((T·H·W)²) 降至 O(T² · poly(H,W))，大幅减少计算量，同时保留帧间全局依赖。
② FrameDiT-G（纯 Matrix Attention）：全部时序 Attention 模块替换为 Matrix Attention，适用于大运动幅度视频，牺牲部分局部细节换取全局时序一致性。
③ FrameDiT-H（混合架构）：Matrix Attention（处理帧间全局大运动）与 Local Factorized Attention（处理帧内局部精细纹理）交替堆叠，同时捕获大运动和小运动，兼顾质量与效率。

关键公式

Matrix Attention（简化形式）：
对帧序列 F = [F_1, ..., F_T]，F_t ∈ R^{H×W×C}
Q_t = F_t · W_Q^L （左矩阵投影，R^{H×d}）
K_t = W_K^R · F_t （右矩阵投影，R^{d×W}）
Attn(t, s) = softmax(Q_t · K_s^T / √d) （帧间注意力）

与已有方法的本质区别

传统方法将帧「展平」为 token 序列再做 Attention；Matrix Attention 保留帧的二维结构，通过矩阵乘法隐式建模空间信息，在帧维度上进行全局 Attention。这种「帧级抽象」使得时序建模的粒度从 token 提升到帧，大幅压缩了序列长度。

✨ 贡献点

1
Matrix Attention 的设计——将帧视为矩阵单元进行帧间注意力，从根本上改变了时序建模的粒度，是新颖的架构思路。
2
FrameDiT-H 混合架构实现了 Full 3D Attention 的质量与 Local Factorized Attention 的效率的有机结合，而非简单取其一。

⚠️ 局限性

潜在问题
  • Matrix Attention 的矩阵投影操作对帧内空间结构做了隐式假设，对非规则运动（如快速旋转）的建模能力可能受限。
  • 混合架构（FrameDiT-H）的最优 Matrix/Local Attention 比例可能需要针对不同任务调优。
  • 超长视频（100+ 帧）场景下 T² 的帧间 Attention 复杂度仍可能成为瓶颈。
