AdaSpark: Adaptive Sparsity for Efficient Long-Video Understanding

Handong Li, Zikang Liu, Longteng Guo et al. · 2026-04

视频理解 · 自适应稀疏 Video-LLM · 长视频 · 时空立方体 arXiv: 2604.08077 ↗

TL;DR

长视频 Video-LLM 处理代价极高。AdaSpark 将视频划分为 3D 时空立方体，通过自适应立方体选择注意力（AdaS-Attn）和可逆信息聚合双组件，动态决定各立方体计算强度，FLOPs 减少 40%+ 同时保持精度。

研究背景

视频 LLM（Video-LLM）在长视频理解上面临严峻的计算挑战：1 分钟视频（24fps）涉及 1440 帧，即使每帧 256 个 visual token 也有近 370k token，对 attention 复杂度和 GPU 内存都是极大压力。

现有效率方法的缺陷：

  • 不可逆信息丢弃（如帧采样、token 压缩）：一旦丢弃就无法恢复，细粒度感知（快速动作、字幕等）受损
  • 固定稀疏模式（如每 N 帧取 1 帧）：无法适应视频内容的动态时序结构，忽视关键片段

核心方法

3D 时空立方体划分：将视频在时间（T）、高度（H）、宽度（W）三个维度划分为非重叠的立方体（如 4×8×8 的时空块），每个立方体是计算分配的基本单元。

双组件协同设计：

  • AdaS-Attn（自适应立方体选择注意力）：
    • 用轻量门控网络评估每个立方体与当前 query 的相关度分数
    • 高相关立方体：分配完整计算（全分辨率 attention）
    • 低相关立方体：降采样后计算（1/4 分辨率）或直接跳过
    • 计算强度动态可变，而非二元"保留/丢弃"
  • 可逆信息聚合：
    • 对被降采样处理的立方体，保留其高层语义摘要而非永久丢弃原始信息
    • 当后续 query 需要时，可恢复（软查询）被降采样立方体的详细信息
    • 关键信息不被永久丢失

实验结果

  • EgoSchema（长视频问答）：准确率 72.4% vs 全量 73.1%，FLOPs 减少 43%
  • Video-MME（综合视频理解）：平均分 65.8 vs 全量 66.3，推理延迟降低 38%
  • MLVU（超长视频）：精度 63.2%（全量 63.8%），支持 1h+ 视频直接推理
  • 动作密集片段检测：AdaS-Attn 自动为动作区域分配更多计算，可视化验证与人类直觉一致

核心贡献

  1. 提出 3D 时空立方体作为统一的自适应计算分配单元，兼顾时间和空间维度
  2. AdaS-Attn 实现连续强度自适应（而非二元选择），保证关键区域细粒度感知
  3. 可逆聚合机制消除不可逆信息丢失，在效率和完整性间达到新 Pareto 前沿

局限性

  • 门控网络引入少量额外参数和训练开销
  • 可逆恢复机制增加内存占用（保留摘要信息），极端内存受限场景下效益降低
  • 立方体划分超参数对性能有影响，需要针对不同视频类型调整
