Data, Not Model: Explaining Bias toward LLM Texts in Neural Retrievers

数据而非模型：解释神经检索器对 LLM 文本的偏好
Wei Huang, Keping Bi, Yinqiong Cai, Wei Chen, Jiafeng Guo, Xueqi Cheng
2026-04-07
arxiv 2604.06163 ↗
TL;DR 检索器偏好 LLM 生成文本的根源在训练数据偏差（正/负样本的流畅度、术语特异性差异），而非模型本身固有缺陷。提出两种有效去偏方案。

📌 研究背景与动机

近期研究发现神经检索器普遍存在来源偏差（source bias）：即便语义相似，也倾向于优先检索 LLM 生成的文段而非人类撰写的文段。这被认为是检索器的固有缺陷，引发了对信息获取系统公平性和可靠性的广泛担忧。本文挑战了这一观点。

🔍 核心发现

偏差来源定位：检索训练数据集中正负样本之间存在系统性的非语义差异（流畅度、术语特异性等），这些差异与 LLM 和人类文本的差异高度重合。
Embedding 空间分析：负→正样本方向与"人类撰写→LLM 生成"文本方向在 embedding 空间高度对齐。
理论证明：检索器在对比学习过程中不可避免地吸收训练数据中的这些人为差异，导致对 LLM 文本的系统性偏好。

⚙️ 去偏方案

方案一：数据清洗 — 减少训练数据中正负样本的人为非语义差异，从根源消除偏差。
方案二：Embedding 去偏 — 在 embedding 空间中减去 LLM 文本向量在偏差方向上的投影，后处理校正。

两种方案均大幅降低来源偏差，验证了偏差源于数据而非模型的核心论断。
