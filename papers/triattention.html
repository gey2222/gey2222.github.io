已有方法的局限性

  • 精度损失严重：H2O、SnapKV 等方法在长推理链场景（CoT 32K+ tokens）下精度急剧下降，因推理中间步骤 token 均关键，不可随意丢弃。
  • 先算后压：现有基于注意力分数的 KV 重要性估计，需要先完成完整注意力计算才能决策哪些 KV 可压缩，无法真正减少计算量。
  • 位置感知建模缺失：已有方法对 Key 的位置依赖性建模不足，缺乏理论支撑的重要性度量，本质上是启发式规则。

本文核心问题

在 32K+ token 长推理场景中，KV Cache 随上下文线性增长，成为吞吐瓶颈。如何在不显著降低精度的前提下，用理论有保证的方式大幅压缩 KV Cache？

⚙️ 核心方法

整体框架

TriAttention 的核心是一个在 pre-RoPE 空间建立的轻量级重要性预测器。关键洞察：Q/K 向量在应用旋转位置编码之前，分布集中在固定非零中心附近（Q/K concentration），且这种集中性跨位置稳定。利用此特性，可以用三角级数拟合 Q/K 中心内积随相对位置距离的变化，从而在不计算完整注意力矩阵的情况下预测任意 Key 的重要性。

关键模块

① Q/K Concentration 分析：在 pre-RoPE 空间统计 Q/K 向量的分布中心 $\bar{q}, \bar{k}$，证明向量集中在非零均值附近，且跨序列位置稳定——这是后续三角建模的几何基础。
② 三角级数位置重要性估计：Q/K 中心向量的内积随相对位置距离 $\Delta p$ 呈周期性变化，可用三角函数级数展开拟合：通过中心向量预测当前 Query 对各位置 Key 的偏好，无需显式注意力计算。
③ Norm 加权 KV 压缩：将三角估计的位置重要性与 Q/K 向量范数相结合，得到综合重要性排名，动态保留 top-K 重要 KV 对，丢弃低重要性历史 KV。

关键公式

Key 重要性估计：
s(p) ≈ ‖q‖ · ‖k‖ · Σₙ aₙ cos(n·Δp + φₙ)
其中 aₙ, φₙ 为三角级数系数，由 Q/K 中心向量的内积分解得到；Δp 为相对位置距离

与已有方法的本质区别

H2O/SnapKV 依赖已计算的注意力分数进行事后 KV 淘汰；TriAttention 在注意力计算之前就可以预测 Key 重要性，利用的是 Q/K 的全局几何结构（中心集中性 + 范数），而非局部注意力分布。这是本质上的范式转变，而非工程优化。

潜在问题
  • 三角级数系数的估计可能对模型架构和预训练数据分布敏感，迁移到新架构模型时需重新校准。
  • Q/K concentration 假设在所有层和任务类型上是否普遍成立尚待验证，对非数学推理任务（如开放生成）泛化性存疑。
